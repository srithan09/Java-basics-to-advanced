Java Access Modifiers

Access Modifier Where it can be accessed Properties When to Use
public Anywhere (any class, any package)
  • Most open access
  • No restrictions
 When you want the method, variable, or class to be accessible from anywhere (like utility methods or APIs).
protected Within the same package + subclasses (even if subclasses are in different packages)
  • Accessible to child classes
  • Slightly more restrictive than public
 When you want to allow controlled inheritance but not complete open access.
default (no keyword) Only within the same package
  • Also called package-private
  • No keyword needed
 When classes or members should be used only by other classes in the same package.
private Only within the same class
  • Most restrictive
  • Not visible to any other class
 When you want to fully hide implementation details. (Use for data hiding, like fields or helper methods.)
